About STRANDS

STRANDS is a simple Web content management system made by type(ish). The focus is on a unique way of visualizing content, and to a lesser extent typography. Everything within the blog or journal is organized in the tapestry. Instead of tags, there are strands, essentially topics. Each post, or knot, is at the intersection of two strands, the primary or A strand and the secondary or B strand. The use of only two strands for each knot is intended to focus the topics of the content. Instead of creating a flood of tags, the writer must decide what the post is specifically about, within at most two categories. This encourages constrained and concise writing.

Visit the STRANDS project page for more information, or contact us at contact@typeish.net.

Using STRANDS

Add and manage content and users through the admin interface. By default, this is located at http://hostname/admin/. Log in using the user you created when syncing the database.

The main page by default shows the three latest knots. All strands can be viewed on the strands page. From there individual knots can be viewed. Also, knots can be viewed by author on the author's page (http://hostname/author/username).

The latest development code of STRANDS can be found at the project's Github repository.
